# 운세 팩트체크 (fortune-check) — 미니앱 기획서

> 한 줄 컨셉: **아침엔 오늘의 운세를 받고, 밤엔 '실제로 맞았는지' O/X로 검증해서 나만의 운세 적중률 통계를 쌓는 운세 검증 앱.**
>
> displayName: **운세 팩트체크** · slug: `fortune-check` · 타깃: 20~30대(전 세대 호환) · 백엔드: Supabase(전국 띠별 랭킹만, 익명)

## 0. 비게임 코어 & 로그인 정책 (1순위 제약 확인)

- **비게임 코어 동사 = 기록·진단·통계.** 코어 루프는 "운세를 **기록**하고, 맞았는지 **진단(O/X)**하고, 적중률 **통계**를 쌓는" 것. 승부/조작/육성/클리어 없음 → **게임 아님**.
- 게이미피케이션(연속 검증 기록·적중률 등급·띠별 랭킹·공유 칭호)은 비게임 코어 위에 얹는 **참여 레이어**일 뿐.
- **로그인 = 없음(익명/로컬 기본).** 토스 로그인 안 씀. 전국 랭킹 집계용으로만 **Supabase 익명 인증** 사용(개인정보 0). 생년월일은 **기기 로컬(localStorage)** 에만 저장하고 서버로 보내지 않음 → **연결끊기 엔드포인트 불필요**.
- 서버에는 익명 식별자 + `{날짜, 띠, 카테고리, O/X}` 집계값만 — 개인 식별 정보 없음.

## 1. 코어 루프 (데일리 훅 · 하루 2세션)

```
[아침] 푸시 "오늘의 운세 도착"
   → 종합 운세 확인(무료)  → 애정/금전/직장 상세 = 📺 보상형 광고 게이트로 해금
   → (아침에 본 카테고리에만 밤 검증 자격 부여 = 호혜 잠금)
        │
[밤]   푸시 "오늘 운세, 맞았어요?"
   → 본 운세마다 O/X 3초 체크인  → 연속 검증 기록(스트릭) 갱신
        │
   → 내 적중률 통계 갱신 + 전국 띠별 적중률 랭킹 갱신
        │
   → 공유 카드(오늘의 운세 / 월간 적중률) → 공유 시 상세 1회 추가 해금
        │
   ←─────────── 다음날 아침 푸시로 복귀 ───────────
```

- **호혜 잠금**: 아침에 확인한 카테고리만 밤에 검증 가능 → 아침 진입 동기.
- **연속 검증 기록(스트릭)**: 끊기면 0으로 리셋(손실 회피). 위기 리마인드 푸시.
- **셀프 피딩 콘텐츠**: "오늘 가장 잘 맞은 띠 랭킹", "전국 평균 적중률 vs 나" 매일 자동 생성.

## 2. 광고 배치 지도

| 액션 | 광고 | 근거 |
|---|---|---|
| 애정/금전/직장 **상세 운세 해금** | **보상형(rewarded)** — `watchThen` | 매일 유저당 1~3회 자연 발생하는 핵심 인벤토리. 완전 선택형 게이트 |
| 공유 카드 → 상세 1회 추가 해금 | 보상형(선택) | 바이럴 + 광고 결합 |
| 검증 완료 → 통계/랭킹 화면 전환 | **전면(interstitial)** `maybeShow(3)` | 자연 경계, 세션당 ≤1 (빈도 3) |
| 홈/통계 탭 하단 | **배너 1개** | 화면당 1개만 |

> 광고 보상은 전부 **결정적(상세 1개 해금 / +1회)** — 확률↑·리롤·뽑기 없음. 사행성 0.

## 3. 가상 보상 경제 (현금 0)

- **연속 검증 기록(스트릭)**: 일 단위 누적. 7일·14일·30일 마일스톤 칭호.
- **적중률 등급**: 누적 검증 수 + 정답률로 "운세 입문자 → 운세 분석가 → 운세 마스터" 등급 상승(결정적, 확률 없음).
- **칭호/뱃지**: "이번 달 금전운 적중왕", "전국 평균 이상" 등 달성 보상.
- 현금/토스포인트 지급은 광고 수익 안정화 이후로 보류(서버 user-key dedup 필요). **현재는 가상만.**

## 4. 리텐션 · 바이럴

- ① 푸시 2슬롯(아침 운세 / 밤 검증) + 스트릭 위기 리마인드
- ② 연속 검증 스트릭(손실 회피)
- ③ 호혜 잠금(아침 봐야 밤 검증)
- ④ 매일 갱신 띠별 랭킹·전국 평균 콘텐츠
- ⑤ 공유 카드 2종(오늘의 운세 / 월간 적중률) → 공유 시 상세 1회 해금

## 5. 데이터 모델 (Supabase — 전국 랭킹만, 익명)

- **로컬(localStorage)**: `profile`(birthDate, birthTime?, zodiac 띠, starSign 별자리, nickname?), `checks`(date×category×verdict), `streak`, `morningViewed:{date}`.
- **서버(Supabase, 익명 인증)**: `fortune_checks(anon_id, kst_date, zodiac, category, verdict)` — 개인정보 0.
  - RPC `zodiac_ranking(p_date)` → 오늘 띠별 적중률 랭킹.
  - RPC `national_accuracy(p_date)` → 전국 평균 적중률.
- 키 없으면(브라우저/개발) **로컬만으로 동작**(전국 랭킹은 placeholder) — 그레이스풀 디그레이데이션.
- 시간 경계는 **KST(Asia/Seoul)**; 서버 집계는 `now() at time zone 'Asia/Seoul'`.

## 6. 운세 콘텐츠 파이프라인

- MVP: **결정적 생성기**(seed = `날짜×띠×별자리×카테고리`)로 **검증 가능한** 문장 풀에서 선택. 오프라인 동작, 단가 0.
- 검증 가능성 가이드: "오후에 예상 밖 지출이 생길 수 있어요"(O/X 가능) ⭕ / "오늘 운이 좋아요"(검증 불가) ❌.
- 톤: 가능성 화법, 위로·응원, 불안 조장·의료·투자 권유 금지.
- 확장(2차): LLM 주간 배치 생성 → 수동 검수 → 큐 적재.

## 7. 비게임 코어 명시 & 심사 차별점

- **코어 동사 = 기록(O/X)·진단·통계.** "게임 아님" — 승부/조작/육성 없음.
- **토스 네이티브 비중복**: 만보기/행운퀴즈/출석앱테크와 메커니즘 다름(코어 = '운세 적중 여부 기록·통계', 승리조건 없음). "출석" 표현 금지 → "연속 검증 기록".
- **만남/연애/채팅 0**: "애정운"은 운세 관례 표현(사람 연결 기능 전무). 랭킹은 익명 띠 단위.
- **사행성 0**: 금칙어 0, 광고 보상 결정적, 가상 보상만.
