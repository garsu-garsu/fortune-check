# 운세 팩트체크 — 앱인토스 미니앱 기획서 (메인 프로젝트)

> 단계: **메인 — 다작 단계에서 전제 검증 후 착수** · 예상 MVP: **5주** · 심사 리스크: 낮음 · 데일리 훅 현실성: **4/5 (풀 전체 1위)**
>
> 한 줄 컨셉: **아침엔 오늘의 운세를 받고, 밤엔 '실제로 맞았는지' O/X로 검증해서 나만의 운세 적중률 통계를 쌓는 운세 검증 앱.**

## 1. 왜 이 앱인가

- **운세는 앱인토스에서 이미 검증된 최강 비게임 카테고리**: 점신이 입점 후 예상치의 6배 일간 이용자 기록. 시장 전체로도 포스텔러 가입자 900만 명, 이용자 83%가 MZ·75%가 여성 — 타겟(20~30대)과 정확히 일치.
- 기존 운세앱의 빈틈: **"읽고 끝"** — 소비형 콘텐츠라 그날의 관계가 아침에 끝난다.
- 이 앱의 비틀기: **"운세를 읽는 앱"이 아니라 "운세를 검증하는 앱"**.
  1. 밤에 한 번 더 여는 **이중 루프**가 생기고
  2. 적중률 통계라는 **나만의 누적 자산**이 생기고(이탈 비용 상승)
  3. "이번 달 내 운세 적중률 73%", "오늘 운세가 가장 잘 맞은 띠 1위: 말띠" 같은 **공유 가능한 집계 콘텐츠**가 매일 자동 생성된다(셀프 피딩 — 1인 운영에 유리).
- 심사 안전: 만남·채팅·매칭·사행성 요소 전무. 운세는 플랫폼 검증 카테고리.

## 2. 코어 루프 (데일리 훅)

**하루 2세션 구조가 핵심:**

| 시간 | 트리거 | 행동 | 장치 |
|---|---|---|---|
| 아침 7시 | 푸시 "오늘의 운세 도착" | 오늘의 운세 확인 (종합+애정/금전/직장) | **아침에 확인해야 밤 검증 자격이 생김** (호혜 잠금) |
| 밤 9시 | 푸시 "오늘 운세, 맞았어요?" | O/X 3초 체크인 | 검증해야 내 적중률·전국 랭킹이 갱신됨 |

- **연속 검증 스트릭**: 끊기면 손실(듀오링고 공식). 스트릭 위기 푸시.
- 매일 갱신 집계 콘텐츠: "오늘 운세가 가장 잘 맞은 띠/별자리 랭킹", "전국 적중률 평균 vs 나".
- 월간 리포트: "6월 내 운세 적중률 73% — 금전운은 잘 맞고 애정운은 영 아니었음" → 공유 카드.
- 적용된 리텐션 공식: ②스트릭, ③아침 확인=밤 검증 자격(호혜 잠금), ④매일 갱신 콘텐츠, ⑤띠별 랭킹.

## 3. MVP 기능 범위 (5주)

**포함**
- 온보딩: 생년월일(+태어난 시간 선택 입력) 등록 → 띠·별자리·일진 산출
- 오늘의 운세: 종합(무료) + 애정/금전/직장 상세(보상형 광고 게이트)
- 밤 O/X 검증 + 연속 검증 스트릭
- 적중률 통계: 개인(월간, 카테고리별) + 전국(띠별 일간 랭킹)
- 공유 카드 2종: 오늘의 운세 카드 / 월간 적중률 카드
- 푸시 2슬롯(아침 운세, 밤 검증) + 스트릭 위기 리마인드

**제외(2차)**
- 친구와 궁합 운세 (친구 초대 해금 보상으로 예약)
- 프리미엄 신년/월간 사주 리포트(인앱결제)
- 정통 사주 만세력 풀이, 타로
- 검증 코멘트("어떻게 맞았는지" 한 줄) — UGC 모더레이션 필요해지므로 보류

## 4. 운세 콘텐츠 파이프라인 (최대 설계 과제)

운세 텍스트 품질이 낮으면 즉시 이탈 — 점신과의 경쟁에서 밀리는 지점도 여기다.

- **생성 구조**: 사주 요소(일진×띠×별자리 조합) 기반으로 LLM 배치 생성 + 템플릿 변주. 매일 실시간 생성이 아니라 **주 단위 배치 생성 → 수동 품질 검수 → 큐 적재**.
- **검증 가능성 설계가 차별화의 핵심**: "오늘 운이 좋아요" 같은 모호한 문장은 O/X 검증이 안 됨. **구체적이고 검증 가능한 문장**으로 생성 가이드를 잡아야 함 — 예: "오후에 예상 밖의 지출이 생길 수 있어요", "오늘 연락 오는 사람 중 반가운 소식이 있어요".
- 톤 가이드: 단정 아닌 가능성 화법, 불안 조장 금지(건강 흉조·재난 예언류 금지), 위로·응원 톤.
- 콘텐츠 단가: LLM 배치 생성이면 사실상 0원. 품질 미달 시 운세 콘텐츠 제휴/외주 플랜 B.

## 5. 토스 플랫폼 연동

- **스마트발송 2회/일이 코어 루프 그 자체** — 다작 단계(출근난이도)에서 실측한 아침 푸시 열람률·최적 발송 시각을 그대로 적용.
- **공유 리워드**: 월간 적중률 카드 + 오늘의 운세 카드 공유 → 상세 운세 1회 해금. 친구 초대 시 **궁합 운세 해금**(2차 기능을 초대 보상으로).
- **토스포인트 프로모션**: 생년월일 등록(온보딩 완주) 시 소액 — 온보딩 전환 견인. 7일 검증 스트릭 보너스.
- **광고 게이트**: 애정/금전/직장 상세 운세 = 보상형 광고. 다작 단계(오늘의 다수결)에서 실측한 게이트 전환율로 무료/광고 경계 설계.

## 6. 수익화

- **보상형 광고**: 상세 운세 해금(매일 자연 발생하는 핵심 인벤토리 — 유저당 일 1~3회)
- 운세 화면 **배너**
- (2차) **프리미엄 사주 리포트 인앱결제**: 신년 운세, 월간 심층 리포트 — 운세 카테고리의 검증된 고단가 BM
- (2차) 궁합 운세 — 초대 보상과 결제 양쪽으로 활용

## 7. 심사 주의사항

- 운세는 검증된 카테고리지만 **미신 콘텐츠 표현 수위** 관리: 의료·투자 판단을 유도하는 문장 금지("오늘 주식 사세요" 불가), 공포 조장 금지.
- 만남·연애 프레이밍 주의: "애정운"은 운세 관례상 무방하나, **사람을 연결해주는 기능은 절대 없음**을 구조적으로 유지(궁합도 "이미 아는 사람과 함께 보기"로만).
- 생년월일 수집 → 개인정보 처리방침에 수집 목적·보관 명시.

## 8. 기술 설계 메모

- 스택: Granite + React 18 + TDS Mobile + Supabase.
- 스키마 스케치:
  - `profiles`(생년월일, 띠, 별자리 — 돛단배 demographics 가드 패턴)
  - `fortunes`(날짜×세그먼트별 운세 텍스트 큐 — 주간 배치 적재)
  - `fortune_checks`(user_id, date, category, verdict O/X — unique(user_id, date, category))
  - 일간 집계 머티리얼라이즈(띠별 적중률 랭킹 — 매 조회 count 금지)
- 검증 자격(아침 열람 여부)·스트릭·KST 자정 리셋: 돛단배 `sent_count_today` KST 패턴 재사용.
- 재사용 자산: `useAdGate`(상세 운세), `share.ts`, `toss-auth`(+생년월일은 자체 입력), `analytics.ts`, 푸시 Edge Function 패턴, KST SQL.
- 운세 생성 배치: Supabase Edge Function 스케줄 또는 로컬 스크립트로 주 1회 생성→검수→업로드.

## 9. 리스크와 대응

| 리스크 | 대응 |
|---|---|
| 점신 등 강자와 정면 경쟁 | "읽는 앱"이 아니라 "검증하는 앱" 포지셔닝 고수 — 적중률 통계를 첫 화면에 |
| 운세 텍스트 품질 미달 | 주간 배치+수동 검수, 검증 가능 문장 가이드. 미달 시 콘텐츠 제휴 플랜 B |
| 푸시 의존 코어 루프 | 다작 단계 실측치로 사전 검증(착수 조건). 푸시 미동의 유저용 '아침 첫 진입 시 운세 강조' 폴백 |
| 밤 검증 세션 절반만 작동 가설 | 검증 미참여자에게도 다음날 "어제 전국 적중률" 콘텐츠로 가치 제공. 검증률 자체를 핵심 지표로 추적 |
